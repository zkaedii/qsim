# Contributor Guidelines

Thank you for your interest in contributing to the QSim project! In order to maintain a high standard, please follow these rules:  

1. **Fork the repository**: Start by forking the repository to your own GitHub account.
2. **Create a new branch**: Your changes should be made in a new branch based on the relevant branch (usually `main` or `develop`). Name the branch descriptively.
3. **Commit your changes**: Make sure to write clear log messages that summarize your changes.
4. **Submit a pull request**: Once your changes are complete and have been pushed to your own fork, submit a pull request to the original repository.

# Development Setup

To set up your environment for development, follow these steps:

1. **Clone the repository**:
   ```bash
   git clone https://github.com/your-username/qsim.git
   cd qsim
   ```
2. **Install dependencies**: Use your package manager of choice (e.g., pip, npm) to install required dependencies. A `requirements.txt` or `package.json` file should be available in the repo.
3. **Run the application**: Check the repo for any instructions on how to run the application locally.

# Pull Request Process

When your feature or bug fix is ready, follow these steps to create a pull request:

1. **Push your changes**: Ensure all changes have been committed and then push your branch to your fork:
   ```bash
   git push origin your-branch-name
   ```
2. **Open a pull request**: Go to the original repository and select your branch and click on the `New Pull Request` button.
3. **Describe your changes**: Please provide a clear description of what your changes do and reference any issues they address.
4. **Address feedback**: Be prepared to make changes as requested by reviewers.

Thank you for helping improve QSim!
